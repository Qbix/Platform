# Qbix Platform — LLM Coding Primer

Comprehensive, authoritative reference for generating correct Qbix code.
Read entirely before writing any PHP, JS, or config.

---

## 1. Plugin Directory Structure

```
MyPlugin/
  config/
    plugin.json                          ← plugin declaration + config
  handlers/
    MyPlugin/
      action/
        post.php                         ← POST MyPlugin/action
        response/
          slotName.php                   ← GET slot "slotName" for MyPlugin/action
  classes/
    MyPlugin.php                         ← static methods class
    MyPlugin/
      ClassName.php                      ← ORM model or subclass
      Base/
        ClassName.php                    ← NEVER edit — autogenerated from DB schema
  web/
    MyPlugin.js                          ← tool registration (required)
    js/tools/toolname.js                 ← Q.Tool.define("MyPlugin/toolname", ...)
    css/tools/toolname.css
  text/
    MyPlugin/content/en.json             ← i18n strings → tool.text.*
  views/
    MyPlugin/content/pagename.php        ← page slot views
    MyPlugin/tool/toolname.php           ← PHP tool renderer (optional)
  scripts/
    MyPlugin/
      1.0-MyPlugin.mysql                 ← DB schema, run by installer
```

---

## 2. PHP Request Dispatch Pipeline

When a request arrives, events fire in this order:

```
Q/prepare   → early hook (rarely used)
Q/validate  → validate input
Q/reroute   → forward() if needed
Q/objects   → create PHP objects
Q/$verb     → handle POST/PUT/DELETE (write to DB here)
Q/analytics → logging
Q/response  → produce output (echo here only)
```

**Only echo/print inside `Q/response` handlers.**  
For POST handlers, Qbix fires `MyPlugin/action/post` automatically — just implement `MyPlugin_action_post()`.

---

## 3. PHP Handlers — Naming Rules

**CRITICAL: action names have NO slashes. One action word only.**

| Request | File | Function |
|---------|------|----------|
| POST `MyPlugin/vote` | `handlers/MyPlugin/vote/post.php` | `MyPlugin_vote_post()` |
| GET slot `things` for `MyPlugin/items` | `handlers/MyPlugin/items/response/things.php` | `MyPlugin_items_response_things()` |
| Page `content` for `MyPlugin/page` | `handlers/MyPlugin/page/response/content.php` | `MyPlugin_page_response_content()` |

**Wrong:** `handlers/MyPlugin/argument/vote/post.php` — slash in action name.  
**Right:** `handlers/MyPlugin/vote/post.php`

Response slot handlers **return** the value — that fills the slot automatically:
```php
function MyPlugin_items_response_things() {
    return array('foo' => 'bar');   // no Q_Response::setSlot needed
}
```

POST handlers use `Q_Response::setSlot` after doing work:
```php
function MyPlugin_vote_post() {
    $user = Users::loggedInUser(true);            // throws if not logged in
    $id   = Q_Request::required('id');            // throws if field missing
    $val  = Q::ifset($_POST, 'value', 1);         // optional with default
    // ... do work ...
    Q_Response::setSlot('result', array('ok' => true, 'id' => $id));
}
```

## 4. PHP Events System

```php
// Fire an event (calls handlers/MyPlugin/doThing.php → MyPlugin_doThing())
Q::event("MyPlugin/doThing", compact('param1', 'param2'));

// Pure event — only runs hooks, no required handler
Q::event("MyPlugin/coolEvent", $params, 'before');   // before hooks only
Q::event("MyPlugin/coolEvent", $params, 'after');    // after hooks only
Q::event("MyPlugin/coolEvent", $params, true);       // both hooks

// Check if handler exists
if (Q::canHandle("MyPlugin/doThing")) { ... }
```

Register hooks in `config/plugin.json`:
```json
{
  "Q": {
    "handlersBeforeEvent": {
      "Streams/post/MyPlugin/thing": ["MyPlugin/before/Streams_post_MyPlugin_thing"]
    },
    "handlersAfterEvent": {
      "Streams/Stream/fetch/MyPlugin/thing": ["MyPlugin/after/Streams_Stream_fetch_MyPlugin_thing"]
    }
  }
}
```

Hook handler signature — `&$return` to modify return value, return `false` to skip subsequent hooks:
```php
function MyPlugin_before_Streams_post_MyPlugin_thing(&$params, &$return) {
    $stream = $params['stream'];
    if (!$stream->testWriteLevel('post')) return false; // skip
    $return = 'custom value'; // overrides event return
}
```

## 5. Q.req — Client to Server

```javascript
// POST — fires MyPlugin/vote/post on server
Q.req('MyPlugin/vote', ['result'], function(err, res) {
    var msg = Q.firstErrorMessage(err, res && res.errors);
    if (msg) return console.warn(msg);
    var result = res.slots.result;
}, { method: 'POST', fields: { id: '123', value: 1 } });

// GET — fills slot 'things' via response/things.php
Q.req('MyPlugin/items', ['things'], function(err, res) {
    var things = res.slots.things;
}, { fields: { filter: 'recent' } });
```

`Q.firstErrorMessage(err, res && res.errors)` — always use this for error checking.

## 6. Routes — config/app.json

```json
{
  "Q": {
    "routes": {
      "claim/:id": { "module": "MyPlugin", "action": "claim" },
      ":module/:action": {}
    }
  }
}
```

Pattern syntax:
- `:variable` — matches any single segment
- `:variable.literal` — matches `anything.literal`
- `:variable[]` — matches any number of remaining segments (last only)
- Literal values must match exactly
- Routes checked last-to-first; `routes@start` checked first, `routes@end` last
- `null` value → route is skipped

Unrouting (URI → URL):
```php
$url = Q_Uri::url("MyPlugin/claim " . json_encode(array('id' => $id)));
```

## 7. Output — Slots, Layouts, Views

```php
// Page content handler
function MyPlugin_page_response_content() {
    $stream = Streams::fetchOne(null, $publisherId, $streamName, true);
    return Q::view('MyPlugin/content/page.php', compact('stream'));
}

// PHP tool renderer
function MyPlugin_thing_tool($options) {
    Q_Response::setToolOptions($options);    // exports options to JS constructor
    $stream = Streams::fetchOne(...);
    return Q::view('MyPlugin/tool/thing.php', compact('stream', 'options'));
}

// Render tools in PHP view
echo Q::tool("MyPlugin/thing", array(
    'publisherId' => $publisherId,
    'streamName'  => $streamName
));

// Multiple tools same type — id required
echo Q::tool("MyPlugin/thing", $options, $uniqueId);

// Multiple tool types on one element
echo Q::tool(array(
    'Q/tabs'          => $tabOptions,
    'MyPlugin/list'   => $listOptions
), 'uniqueId');

// Pass data to client JS
Q_Response::setScriptData("MyPlugin.thing", $thing->exportArray());
Q_Response::setScriptData("MyPlugin.things", Db::exportArray($things));

// Preload a stream (client gets it cached on page load)
$stream->addPreloaded($asUserId);

// Add scripts/stylesheets to slot
Q_Response::addScript('js/foo.js');
Q_Response::addStylesheet('css/foo.css');

// Export view as client-side Handlebars template
Q_Response::addTemplate("MyPlugin/thing", "handlebars");

// Redirect (before Q/response fires)
Q_Response::redirect('MyPlugin/welcome');

// Errors for form validation (client renders inline)
Q_Response::addError($exception);
```

Q_Html helpers (URI strings or URLs):

## 8. Q.Tool.define — Complete Pattern

```javascript
(function ($, window, undefined) {

Q.Tool.define("MyPlugin/name",

// ── 1. Constructor ────────────────────────────────────────────────────────
function (options) {
    var tool  = this;
    var state = tool.state;
    // tool.text.*  — i18n strings from en.json (loaded via text: [...] in registration)

    tool.rendering(['fieldA', 'fieldB'], function (changed, previous) {
        // DO NOT read from DOM here. DO NOT modify tool.state here.
        if (tool.$someEl) tool.$someEl.textContent = state.fieldA;
    });

    tool.refresh();
},

// ── 2. Default options ────────────────────────────────────────────────────
{
    publisherId: null,
    streamName:  null,
    fieldA:      '',
    editable:    false,
    onRefresh:   new Q.Event()
},

// ── 3. Methods ────────────────────────────────────────────────────────────
{
    refresh: function () {
        var tool  = this;
        var state = tool.state;

        Q.Template.render('MyPlugin/name', {
            text:   tool.text,
            fieldA: state.fieldA
        }, function (err, html) {
            if (err) return console.warn(err);
            tool.element.innerHTML = html;
            Q.activate(tool.element, function () {
                tool.$someEl = tool.element.querySelector('.MyPlugin_some');
                tool._bindEvents();
                state.onRefresh.handle.call(tool);
            });
        });
    },

    _bindEvents: function () {
        var tool = this;
        $(tool.element).on('click.MyPlugin_name', '.MyPlugin_btn', function () {
            tool.state.fieldA = 'new value';
            tool.stateChanged('fieldA');
        });
    },

    _addChild: function (publisherId, streamName) {
        var tool = this;
        var id   = streamName.split('/').pop();
        var el   = Q.Tool.setUpElement(
            'div',
            'MyPlugin/child',
            { publisherId: publisherId, streamName: streamName },
            id,
            tool.prefix   // ALWAYS pass parent prefix
        );
        tool.element.querySelector('.MyPlugin_list').appendChild(el);
        Q.activate(el);
    },

    Q: {
        beforeRemove: function () {
            $(this.element).off('.MyPlugin_name');
        },
        onRetain: function (newOptions) {
            // compare newOptions to this.state and update display
        },
        onInit: function () {
            // called after all child tools are initialized
        }
    }
});

Q.Template.set('MyPlugin/name',
    '<div class="MyPlugin_some">{{fieldA}}</div>' +
    '<button class="MyPlugin_btn">{{text.DoSomething}}</button>'
);

})(window.jQuery, window);
```

**Key rules:**
- `tool.rendering(fields, fn)` — deferred to animation frame; don't read DOM or modify state inside
- `tool.stateChanged('field1,field2')` — signal change after modifying state
- `tool.$('selector')` — jQuery scoped to tool element
- `Q.Tool.clear(container)` — removes child tools before replacing innerHTML
- `Q.Tool.setUpElement('div', 'Plugin/child', opts, id, tool.prefix)` — always pass `tool.prefix`
- Child IDs must be unique among siblings — use stream's last path segment
- Pass tool reference (not string key) to `event.set(handler, tool)` — auto-removes on tool removal

## 11. Database — Db Layer

```php
$db = Db::connect('MyPlugin');

// SELECT
$rows = $db->select('*', 'myplugin_thing')
    ->where(array(
        'publisherId' => $publisherId,   // = operator prepended automatically
        'type'        => 'MyPlugin/claim',
        'status !='   => 'deleted',      // operator in key
        'score >'     => 5
    ))
    ->orderBy('score', false)            // false = DESC
    ->limit(20, 0)                       // limit, offset
    ->fetchAll(PDO::FETCH_ASSOC);

// IN clause
$rows = $db->select('*', 'myplugin_thing')
    ->where(array('type' => array('type1', 'type2', 'type3')))
    ->fetchAll();

// Db_Range for inequalities
$range = new Db_Range($min, true, false, $max);   // $min <= x < $max

// Db_Expression — raw SQL, bypasses sanitization
->where(array('created >' => new Db_Expression("NOW() - INTERVAL 1 DAY")))

// INSERT
$db->insert('myplugin_thing', array(
    'id'          => $id,
    'publisherId' => $publisherId,
    'content'     => $content
))->execute();

// UPDATE
$db->update('myplugin_thing')
    ->set(array('score' => 10, 'status' => 'active'))
    ->where(array('id' => $id))
    ->execute();

// DELETE
$db->delete('myplugin_thing')->where(array('id' => $id))->execute();

// Batch insert with upsert
MyPlugin_Thing::insertManyAndExecute($rows, array(
    'onDuplicateKeyUpdate' => array('score' => 1)
));
```

Fetching via query caches by SQL — same query twice in one request only hits DB once.

## 12. Models (ORM — Db_Row)

Edit `classes/MyPlugin/Thing.php`, **never** `classes/Base/`.

```php
// Insert
$thing = new MyPlugin_Thing();
$thing->id          = MyPlugin_Thing::db()->uniqueId(MyPlugin_Thing::table(), 'id');
$thing->publisherId = $publisherId;
$thing->content     = $content;
$thing->save();                            // INSERT

// Fetch one
$thing = new MyPlugin_Thing();
$thing->id = $id;
if (!$thing->retrieve()) {
    throw new Q_Exception_MissingRow(array('table' => 'thing', 'criteria' => $id));
}

// Upsert (ON DUPLICATE KEY UPDATE)
$thing = new MyPlugin_Thing();
$thing->id = $id;
$thing->save(true);

// Fetch many
$things = MyPlugin_Thing::select('*')
    ->where(array('publisherId' => $publisherId))
    ->orderBy('insertedTime', false)
    ->limit(20, 0)
    ->fetchDbRows();   // returns array of MyPlugin_Thing objects

// Update (after retrieve)
$thing->score = 10;
$thing->save();    // UPDATE

// Delete
$thing = new MyPlugin_Thing();
$thing->id = $id;
$thing->remove();

// In-memory metadata (not persisted)
$thing->set('key', $value);
$val = $thing->get('key', $default);
$thing->clear('key');

// Export to client
Q_Response::setScriptData("MyPlugin.thing", $thing->exportArray());
Q_Response::setScriptData("MyPlugin.things", Db::exportArray($things));

// Hooks
function beforeSave($modifiedFields) {
    // validate or transform before save
    return parent::beforeSave($modifiedFields);
}
function afterSaveExecute($result, $query, $modifiedFields, $where) {
    // fires after INSERT/UPDATE — use for cascades, notifications
    return $result;
}

// Relations between models (in setUp())
$this->hasOne('author', array('a' => 'MyPlugin_Thing', 'u' => 'Users_User'),
    array('a.byUserId' => 'u.id'));
$this->hasMany('tags', array('a' => 'MyPlugin_Thing', 't' => 'MyPlugin_Tag'),
    array('a.id' => 't.thingId'));
// Usage: $thing->get_tags('*');
```

Class autoloading: `MyPlugin_Thing_Sub` → `classes/MyPlugin/Thing/Sub.php`.

## 13. Streams — PHP

```php
// Fetch
$stream = Streams::fetchOne($asUserId, $publisherId, $streamName);          // null if missing
$stream = Streams::fetchOne($asUserId, $publisherId, $streamName, true);    // throws if missing
$stream = Streams::fetchOneOrCreate($asUserId, $publisherId, $streamName, array(
    'fields'     => array('type' => 'MyPlugin/thing', 'title' => 'Title'),
    'skipAccess' => true
), $results);
if ($results['created']) { /* first time */ }

// Create
$stream = Streams::create($asUserId, $publisherId, array(
    'type'       => 'MyPlugin/thing',
    'title'      => 'Title',
    'content'    => 'Description',
    'attributes' => Q::json_encode(array('foo' => 'bar'))
));

// Attributes
$val = $stream->getAttribute('foo', $default);
$stream->setAttribute('foo', 'bar');
$stream->setAttribute(array('foo' => 'bar', 'baz' => 1));  // batch
$stream->changed($asUserId);                                // save + notify + fire hooks

// Access checks
$stream->testReadLevel('content');    // 'none','see','content','participants','messages'
$stream->testWriteLevel('post');      // 'none','join','vote','post','relate','edit','close'
$stream->testAdminLevel('manage');    // 'none','tell','invite','manage','own'
$stream->testPermission('myPerm');

// Post message
$stream->post($asUserId, array(
    'type'         => 'MyPlugin/thing/updated',
    'content'      => 'Human-readable description',
    'instructions' => Q::json_encode(array('key' => 'value'))
), true);   // true = skip access check

// Relate (fromStream = child, toStream = parent/category)
Streams::relate(
    $asUserId,
    $toPublisherId, $toStreamName,    // parent
    'MyPlugin/relationType',
    $fromPublisherId, $fromStreamName, // child
    array('weight' => 1)
);
Streams::unrelate($asUserId, $toPublisherId, $toStreamName,
    'MyPlugin/relationType', $fromPublisherId, $fromStreamName);

// Fetch related
list($relations, $relatedStreams, $stream) = Streams::related(
    $asUserId,
    $publisherId,
    $streamName,
    'MyPlugin/relationType',
    true,              // isCategory=true → fetch "from" streams (children)
    array('limit' => 50)
);

// Cheap relations query (no stream fetch)
$rows = Streams_RelatedTo::select('*')->where(array(
    'toPublisherId' => $publisherId,
    'toStreamName'  => $streamName,
    'type'          => 'MyPlugin/relationType'
))->fetchDbRows();
// each row: ->fromPublisherId, ->fromStreamName, ->type, ->weight

// Join / subscribe
$stream->join($asUserId);        // user becomes participant
$stream->subscribe($asUserId);   // user gets offline notifications

// Preload for client
$stream->addPreloaded($asUserId);
```

**Stream naming:** `name` always starts with type prefix:  
`type: "MyPlugin/claim"` → `name: "MyPlugin/claim/{id}"`

## 14. Streams — JavaScript

```javascript
// Fetch one stream
Q.Streams.get(publisherId, streamName, function (err, stream) {
    if (err) return console.warn(err);
    var val   = stream.getAttribute('foo');
    var title = stream.fields.title;    // always .fields.name NOT .streamName
    // fields: publisherId, name, type, title, content, attributes, messageCount, ...
});

// Retain for tool lifetime
Q.Streams.retainWith(tool).get(publisherId, streamName, function () {
    var stream = this;
    tool.state.stream = stream;
});

// Fetch related streams
// CRITICAL: callback is function(errorMessage) — NOT function(err, result)
Q.Streams.related(
    publisherId, streamName,
    'MyPlugin/relationType',
    true,              // isCategory: true → children are "from"
    { limit: 50 },
    function (errorMessage) {
        if (errorMessage) return console.warn(errorMessage);
        var result = this;
        Q.each(result.relations, function () {
            var child = this.from;   // isCategory=true → use .from
        });
        // result.relatedStreams — keyed "publisherId\tstreamName"
        // result.stream — the parent stream
    }
);

// Live message listener (tied to tool lifetime)
stream.onMessage('MyPlugin/thing/updated').set(function (msg) {
    var inst = JSON.parse(msg.instructions || '{}');
    // msg: ordinal, type, content, instructions, byUserId, sentTime
}, tool);

// Listen for any message on a stream type
Q.Streams.onMessage('MyPlugin/thing', 'MyPlugin/thing/updated').set(handler, key);

// Stream field access
stream.fields.name          // stream name (NOT .streamName)
stream.fields.publisherId
stream.fields.type
stream.fields.title
stream.fields.content
stream.getAttribute('key')  // from JSON attributes
stream.setAttribute('key', value);   // pending; call stream.save() to persist
stream.save(callback);       // issues PUT to Streams/stream

// Field/attribute change events
Q.Streams.Stream.onUpdated(pubId, streamName, 'attrName').set(function (attrs) { ... });
Q.Streams.Stream.onFieldChanged(pubId, streamName, 'title').set(function (fields) { ... });

// Refresh all retained streams (call after coming back online)
Q.Streams.refresh(callback);

// Define a client-side stream type constructor
Q.Streams.define("MyPlugin/thing", function (fields) {
    this.fields.extra = fields.extra;
    this.onMessage('MyPlugin/thing/updated').set(function (stream, msg) {
        // update local stream state from message
    }, 'MyPlugin/thing');
    // DO NOT add events as properties here — stream objects are replaced on refresh
});
```

## 15. Streams — Access Control

Access levels are numeric; use string constants when testing:

| Level | Read | Write | Admin |
|-------|------|-------|-------|
| 0 | none | none | none |
| 10 | see | join | publish |
| 13 | — | vote | — |
| 15 | teaser | suggest | — |
| 18 | — | contribute | — |
| 20 | relations | post | invite |
| 23 | content | relate | — |
| 25 | fields | — | — |
| 30 | participants | edit | manage |
| 35 | messages | — | — |
| 40 | receipts | close | own |

**Common test strings:** `testReadLevel('messages')` = 35, `testWriteLevel('post')` = 20, `testWriteLevel('relate')` = 23, `testWriteLevel('edit')` = 30, `testAdminLevel('manage')` = 30, `testAdminLevel('own')` = 40.

Public access: `readLevel`, `writeLevel`, `adminLevel` fields on `Streams_Stream` row.  
Granular: `Streams_Access` rows with `ofContactLabel` (aggregated by max) or `ofUserId` (overrides labels).  
Templates: stream with `name = "$type/"` — fields/access copied to new streams of that type.  
Mutable: stream with `name = "$type/*"` — access computed at fetch time, not copied.

Stream type config in `plugin.json`:
```json
{
  "Streams": {
    "types": {
      "MyPlugin/thing": {
        "create": ["content", "title"],
        "edit":   ["content", "title"],
        "defaults": {
          "readLevel": 40,
          "icon": "MyPlugin/thing"
        },
        "messages": {
          "MyPlugin/thing/updated": {
            "edit": true,
            "subject": "Thing was updated"
          }
        }
      }
    }
  }
}
```

## 17. Assets / Credits

```php
// Unified payment — never throws; returns array with 'success' key
$result = Assets::pay(
    $communityId,
    $userId,
    1,                         // amount (credits by default)
    'MyPlugin/reason',         // reason — required, appears in ledger
    array(
        'currency'      => 'credits',
        'toPublisherId' => $publisherId,
        'toStreamName'  => $streamName
    )
);
if (!$result['success']) {
    Q_Response::setSlot('result', array(
        'success'     => false,
        'needCredits' => Q::ifset($result, 'details', 'needCredits', 1),
        'intentToken' => Q::ifset($result, 'details', 'intentToken', null)
    ));
    return;
}

// Transfer credits between users
Assets::transfer($communityId, $amount, 'MyPlugin/reason',
    $toUserId, $fromUserId, array('forcePayment' => true));

// Direct spend/grant
Assets_Credits::spend($communityId, $amount, $reason, $userId, $options);
Assets_Credits::grant($communityId, $amount, 'ReasonKey', $userId);

// Check balance
```

## 20. JS Patterns (Q.pipe, Q.getter, Q.latest, Q.debounce)

```javascript
// Q.pipe — run handler when multiple async results arrive
var p = new Q.Pipe(['user', 'stream'], function (args, subjects) {
    // args.user, args.stream — arguments passed to fill()
    // subjects.user, subjects.stream — "this" context
});
Q.Users.get(userId, p.fill('user'));
Q.Streams.get(pubId, name, p.fill('stream'));

// Chaining pipes (second param true = don't re-run this step)
var p2 = new Q.Pipe(['users', 'stream'], function (args, subjects) {
    doSomething(args.users, args.stream);
    db.query(p2.fill('extra', true));  // true = don't re-trigger this step
}).set(['users', 'extra'], function (args) {
    // runs when both 'users' and 'extra' are filled
});

// Q.latest — ignore stale async responses
var ordinal = Q.latest('MyPlugin/search');
doAsyncRequest(function (err, results) {
    if (!Q.latest('MyPlugin/search', ordinal)) return; // stale
    renderResults(results);
});

// Q.debounce — fire after N ms of silence
$(input).on('keyup', Q.debounce(function() { search(); }, 300));

// Q.throttle — fire at most once per N ms
$(window).on('scroll', Q.throttle(function() { update(); }, 100));

// Q.getter — caching, throttling, waitlisting wrapper
MyPlugin.getThings = Q.getter(function (id, callback) {
    Q.req('MyPlugin/things', ['result'], function(err, res) {
        callback(err, res && res.slots.result);
    }, { fields: { id: id } });
}, {
    cache: new Q.Cache.document("MyPlugin.getThings"),
    throttle: "MyPlugin.getThings"
});

// Q.once — call at most once
getVariousResults(Q.once(callback));

// Q.normalize — standard identifier chars
var id = Q.normalize(name);
```

## 23. Streams Stream Naming & Ontology

Stream name always starts with type prefix:
```
type: "MyPlugin/claim"    → name: "MyPlugin/claim/{id}"
type: "MyPlugin/argument" → name: "MyPlugin/argument/{id}"
type: "Assets/credits"    → name: "Assets/credits/{userId}"
```

External publisher IDs: reverse-domain — `com.twitter`, `com.facebook`.  
Relations: **fromStream = child, toStream = parent/category**.  
`isCategory=true` = fetching children (from) of this parent (to).

Built-in stream ontology (`Streams/`): text, file, image, audio, video, pdf, live, chat,
category, task, tasks, interest, topic, question, answer, search, space.  
(`Calendars/`): calendar, event. (`Assets/`): product, service, holding.

## 25. Common Mistakes

| Wrong | Right |
|-------|-------|
| `handlers/MyPlugin/argument/vote/post.php` | `handlers/MyPlugin/vote/post.php` — no slashes in action name |
| `function MyPlugin_vote_POST()` | `function MyPlugin_vote_post()` — lowercase |
| `Q.Streams.related(..., function(err, related){})` | `function(errorMessage){ var result = this; }` |
| `this.to` when `isCategory=true` | `this.from` — use `.from` for children |
| `stream.fields.streamName` | `stream.fields.name` — always `.name` |
| `Q.Tool.setUpElement(..., id, null)` | Always pass `tool.prefix` as last arg |
| `Q.Tool.setUpElement` without `Q.activate` | Always follow with `Q.activate(el)` |
| Raw innerHTML replace with child tools inside | `Q.Tool.clear(container)` first |
| jQuery events without namespace | `.on('click.MyPlugin_name', ...)` always namespace |
| No `.off('.MyPlugin_name')` in beforeRemove | Always clean up namespaced events |
| `Q_Response::setSlot` in response slot handler | `return $value` — return fills slot |
| `$stream->save()` to persist attributes | `$stream->setAttribute(...); $stream->changed($userId)` |
| Editing `classes/Base/...` files | Only edit `classes/MyPlugin/...` |
| Calculating totals manually | Use `Users_Total` / `Assets_Badge::badgesAndTotals()` |
| Adding events as properties in `Q.Streams.define` | Stream objects replaced on refresh — don't do `this.onFoo = new Q.Event()` |
| `new Q.Pipe(...)` without calling `pipe.run()` after adding handlers | Call `p.run()` if objects may already be filled |
| Off-session Stripe charge without saved card | Only works if first checkout used `setup_future_usage = 'off_session'` |
| Firing `Q::event` for side effects inside `Q/response` | Use `Q/$verb` (Q/post, Q/put) for state changes; `Q/response` is output only |

## 27. Schema-Derived Corrections to Common Patterns

```php
// streams_access: -1 means "don't override this level"
// To ban a specific user from reading but leave write alone:
$access = new Streams_Access();
$access->publisherId = $publisherId;
$access->streamName  = $streamName;
$access->ofUserId    = $bannedUserId;
$access->readLevel   = 0;   // none
$access->writeLevel  = -1;  // unchanged
$access->adminLevel  = -1;  // unchanged
$access->save(true);

// Correct field: referredByUserId (not byUserId or invitingUserId)
$r = new Users_Referred();
$r->userId           = $userId;
$r->toCommunityId    = $communityId;
$r->referredByUserId = $invitingUserId;

// streams_participant state values
// 'invited' | 'participating' | 'left'  (not 'joined')
$p = Streams_Participant::select('*')
    ->where(array(
        'publisherId' => $publisherId,
        'streamName'  => $streamName,
        'state'       => 'participating'
    ))->fetchDbRows();

// streams_message: ordinal is unsigned int, auto-set by Streams layer
// Never set ordinal manually — let Streams::post() handle it
// instructions is varchar(8191) — keep JSON under ~8000 chars
// content is varchar(4095) — human-readable summary only

// assets_credits: 'amount' can be negative (debit) or positive (credit)
// Always go through Assets::pay() or Assets_Credits::spend()/grant()
// Never write directly to assets_credits — use the API

// users_total: NEVER UPDATE MANUALLY
// Users_Vote::vote() calls beforeSave() which auto-updates users_total
```
